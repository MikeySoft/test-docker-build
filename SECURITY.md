# Security Policy

We take security seriously and appreciate responsible disclosure of vulnerabilities.

## Reporting a Vulnerability

- Please submit a bug report on github.
- Include detailed steps to reproduce, affected versions, and potential impact.
- Do **not** create a public GitHub issue for security findings.

We will acknowledge receipt within 2 business days and aim to provide an initial assessment within 7 business days.

## Disclosure Process

1. We investigate and confirm the issue.
2. We coordinate a fix and determine CVE requirements if applicable.
3. We notify you of the remediation timeline.
4. We publish a security advisory alongside the patch release.
